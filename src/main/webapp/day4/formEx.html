Form Tag의 개론1

form tag의 기본 속성 알아보기

  • form tag에는 데이터 서버에 전솔할 때 전송 방식을 결정할 수 있다.
  • 이 전송 방식을 method 라는 속성으로 정의하는데, 기본값은 get(데이터 길이제한 있음 @4,100 bytes)이다.
  • method는 약 7개로 값이 정의 되어 있는데, get, post, put, delete, trace,...
  • get은 일반적으로 데이터 조회(select)를 서버에 요청 시에 사용됨
  • post는 데이터 생성(insert), 갱신(update), 삭제(delete)를 서버에 요청시에 사용됨
  • 나머지 것들은 RESTfull Service를 강의할 때 알게됩니다.
  • name 속성 : 폼의 이름을 지정합니다. 자바스크립트나 DOM 등을 이용해서 form 데이터를 제어할 때 사용됨
  • action : 전솔될 서버의 URL을 명기함. 이때 대부분의 서버페이지는 서버프로그래밍이 적용된 페이지
  • enctype : 폼 데이터의 encoding 타입을 명기함. 나중에파일 업로드시에 보기됨
  • target : 폼 데이터가 처리된 이후에 결과를 출력할 프레임이나 창을 지정할 수 있다. anchor의 target과 같음
